🟡 IN PROGRESS · Databases

HQ Databases primitive (Notion parity)

B1 shipped 2026-06-21 (#69): data model + RLS + read-only table view — the substrate to retire Notion's databases (typed tabular records, which don't serialize to markdown and were lost in the page migration). Four org-scoped tables (hq_databases / hq_db_properties / hq_db_records JSONB values / hq_db_entities junction), 9 property types, full RLS up front. New "Databases" tab in the Knowledge hub renders a database's records as a typed table (chips for select/status/multi-select, checkboxes, UTC-safe dates, URL links). A demo database is seeded in prod (safe to delete). B2 shipped 2026-06-21 (#71): the Notion-API importer — an in-app "▤ Import from Notion" panel takes a Notion DB URL + a read-only integration token (entered per-import, not stored, OQ-4) and recreates the schema + every row in HQ. A pure mapping layer (spec §5 type map) handles title/text/select/status/multi-select/number/checkbox/date/url/relation, normalizes select colors, and snapshots formula/rollup/people/files as read-only text (reported back). Writes go through the caller's RLS-scoped client (no service role, no migration — B1's FOR ALL policies already cover it); imports default to entity tm (OQ-6). 23 unit tests. Next: first real import = the recovered Semantic Layer DBs (App Sections 28 rows + Metrics 53 rows) once run with the HQ Importer token; B3 = in-app row/column editing + inline-on-page embeds + HTML/CSV export + cross-DB relation resolution. Spec hq_databases_v1.html (HQ Knowledge 0ed77d44); all 6 OQs locked 2026-06-20.
